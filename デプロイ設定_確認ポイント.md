## ⚠️ 必ず確認してほしいこと

### 1. ビルド構成が「cloudbuild-deploy.yaml」か

トリガーの「構成」で、**cloudbuild-deploy.yaml** を使うように設定されているか確認してください。

- **自動検出**のまま → ❌ `cloudbuild.yaml` が使われ、**イメージのビルドだけ**でデプロイされません
- **Cloud Build 構成ファイル**を選び、`cloudbuild-deploy.yaml` を指定 → ✅ ビルド＋デプロイが実行されます

**確認・修正手順：**

1. [Cloud Build トリガー](https://console.cloud.google.com/cloud-build/triggers) を開く
2. 「nutrition-app-deploy」の **⋮**（3点メニュー）→ **編集**
3. 「構成」セクションで **「Cloud Build 構成ファイル」** を選択
4. 「Cloud Build 構成ファイルの場所」に `cloudbuild-deploy.yaml` と入力
5. **保存**

---

### 2. cron-secret が Secret Manager にあるか

Cloud Shell で「No credentialed accounts」や「You do not currently have an active account selected」が出ていた場合、**cron-secret** がまだ登録されていない可能性があります。

[Secret Manager](https://console.cloud.google.com/security/secret-manager) で以下があるか確認してください：

- `google-credentials`
- `google-token`
- `cron-secret`

**cron-secret がない場合：** 一度 `./deploy.sh` を実行した環境（gcloud が使える Mac や Cloud Shell でログイン済みの環境）で、以下を実行します。

```bash
CRON_SECRET=$(openssl rand -hex 16)
echo -n "$CRON_SECRET" | gcloud secrets create cron-secret --data-file=-
```
