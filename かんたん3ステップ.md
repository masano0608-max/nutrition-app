# かんたん3ステップ（もうわからない人向け）

やることは **3つだけ** です。

---

## ステップ1: スクリプトを Cloud Shell に置く

### 方法A: ドラッグ＆ドロップ（いちばん簡単）

1. **Cloud Shell** を開く  
   https://shell.cloud.google.com/?project=project-646f6956-aa77-4c66-b92

2. 左側の **フォルダアイコン**（エクスプローラー）をクリック

3. パソコンの Finder で、このフォルダを開く：  
   `子供達の栄養管理`

4. **トリガーを修正してシークレット登録.sh** を、Cloud Shell の左のパネルに **ドラッグ＆ドロップ**

5. 下の **黒い画面（ターミナル）** で、次を **1行ずつ** 入力して Enter：
   ```
   cd ~
   chmod +x トリガーを修正してシークレット登録.sh
   ./トリガーを修正してシークレット登録.sh
   ```
   （ファイルが別の場所にある場合は、一覧表示で場所を確認してから `cd` で移動）

---

## ステップ2: GitHub への push は GitHub Desktop で

1. **GitHub Desktop** を開く（入っていなければ https://desktop.github.com からインストール）

2. **File** → **Add Local Repository** で「子供達の栄養管理」フォルダを選ぶ

3. 上部の **「Push origin」** をクリック

（ログインしていれば、これだけで push できます）

---

## ステップ3: トリガーの設定だけ手動で

1. このページを開く：  
   https://console.cloud.google.com/cloud-build/triggers?project=project-646f6956-aa77-4c66-b92

2. **nutrition-app-deploy** の右側の **⋮**（3点）→ **編集**

3. 下の方の「構成」で：
   - **「Cloud Build 構成ファイル」** を選択
   - 入力欄に **`cloudbuild-deploy.yaml`** と入力

4. **保存** をクリック

---

## これで完了

- ステップ1 … トリガー修正とシークレットの準備
- ステップ2 … コードを GitHub に送る
- ステップ3 …  push でデプロイされるようにする

難しかったら、**ステップ2 と 3 だけ** やっても大丈夫です。  
（ステップ1 は Cloud Shell の認証が通る場合に効果があります）
