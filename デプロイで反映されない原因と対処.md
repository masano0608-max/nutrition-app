# デプロイしたのに変更が反映されない - 原因と対処

---

## 根本原因1: 変更が GitHub に push されていない ⚠️ 今回これ

**症状:** コードを編集したが、GitHub に送っていない

**確認方法:**
```bash
cd /Users/masanotanaka/子供達の栄養管理
git status
```
「Changes not staged for commit」や「Changes to be committed」がある = まだ push されていない

**対処:**
1. コミットする:
   ```bash
   git add .
   git commit -m "変更内容の説明"
   ```
2. **push する**（ここが必須）:
   ```bash
   git push origin main
   ```
   - GitHub Desktop を使う場合: 「Push origin」をクリック
   - ブラウザアップロード: https://github.com/masano0608-max/nutrition-app/upload/main

**重要:** Cloud Build は **GitHub の main ブランチ** をクローンしてビルドします。ローカルの変更が GitHub にないと、古いコードでビルドされます。

---

## 根本原因2: トリガーが「ビルドのみ」でデプロイしていない

**症状:** push すると Cloud Build は動くが、Cloud Run が更新されない

**原因:** トリガーが `cloudbuild.yaml`（自動検出）を使っている。このファイルは**イメージをビルドするだけ**で、Cloud Run へのデプロイをしません。

**対処:**
1. [Cloud Build トリガー](https://console.cloud.google.com/cloud-build/triggers) を開く
2. 「nutrition-app-deploy」の **⋮** → **編集**
3. 「構成」で **「Cloud Build 構成ファイル」** を選択
4. ファイルの場所に **`cloudbuild-deploy.yaml`** と入力
5. 保存

---

## 根本原因3: ブラウザのキャッシュ

**症状:** デプロイは成功しているが、古い画面が表示される

**対処:**
- スマホ: アプリのURLを開き直す、または **スワイプで更新**
- PC: **Ctrl+Shift+R**（Mac: **⌘+Shift+R**）で強制リロード
- シークレットウィンドウで開き直す
