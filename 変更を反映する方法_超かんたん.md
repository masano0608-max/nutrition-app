# 変更をアプリに反映する方法（超かんたん）

パソコンで編集した内容を、ネットのアプリに反映させる手順です。

---

## イメージ

```
あなたのパソコン          GitHub（お預けする場所）        Cloud Run（アプリの本番）
    │                              │                              │
    │  ① push（送る）              │                              │
    │ ──────────────────────────►  │                              │
    │                              │  ② 自動で取りに行く            │
    │                              │ ──────────────────────────►  │
    │                              │                              │
    │  編集したファイル             │  お預けしたファイル            │  みんなが使うアプリ
```

**① を忘れると、② で古いままのファイルが使われます。**

---

## やること（1つ）

### 「push」= パソコンの変更を GitHub に送る

これだけできれば、あとは自動でアプリに反映されます。

---

## やり方（いちばん簡単）

### GitHub Desktop を使う

1. **GitHub Desktop** というアプリを開く  
   （入っていない場合 → https://desktop.github.com でダウンロード）

2. 左側に **「子供達の栄養管理」** または **「nutrition-app」** が出ているか確認

3. 出ていない場合：
   - **File** → **Add Local Repository**
   - フォルダ「子供達の栄養管理」を選ぶ
   - **Add Repository** をクリック

4. 画面上部に **「Push origin」** というボタンがあるか確認
   - あれば → それを **1回クリック**
   - 「Fetch origin」とあれば → その右の **「Push origin」** をクリック

5. これで送信完了です。2〜3分待つとアプリに反映されます。

---

## うまくいかないとき

| 状況 | 対処 |
|------|------|
| GitHub Desktop にリポジトリがない | File → Add Local Repository で「子供達の栄養管理」フォルダを選ぶ |
| Push origin が押せない | まず「Commit to main」でコミットしてから、Push origin を押す |
| ログインを求められる | ブラウザが開くので、GitHub にログインする |
| アプリがまだ古い | 2〜3分待つ。スマホならページを下にスワイプして更新する |

---

## ポイント

- **編集したら → 必ず Push**
- Push しないと、アプリには反映されません。
